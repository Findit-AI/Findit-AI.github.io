Real production libraries move across editors, machines, and drives.

If retrieval context breaks on every migration, teams lose hours before editing even starts.

## Migration checklist

Before moving drives:

- confirm indexing is complete for active folders
- keep folder hierarchy unchanged during transfer
- align destination path conventions with collaborators

After reconnecting on another machine:

- verify mount paths
- run a small validation query set
- rebuild only affected indexes when needed

Portable indexing is not a nice-to-have. It is what keeps retrieval operational in multi-device workflows.
